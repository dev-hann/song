# Discovery Domain

## 검색 (Search)

YouTube에서 키워드로 영상 검색.

### 결과

- 각 결과는 id, title, thumbnail, duration, channel 정보 포함
- 연속 검색 지원 (추가 결과 로드 가능)
- 필터: 현재 video 타입만 지원

---

## 멜론 차트 (Melon Chart)

멜론에서 스크래핑한 음악 차트 데이터.

### 차트 종류

| Type | Description |
|------|-------------|
| 실시간 | 멜론 실시간 차트 |
| HOT100 | 멜론 HOT100 차트 |
| 일간 | 멜론 일간 차트 |

### 차트 항목

| Property | Description |
|----------|-------------|
| rank | 순위 |
| title | 곡 제목 |
| artist | 아티스트 |
| album | 앨범명 |
| albumArt | 앨범 아트 URL |

### 정책

- 외부 스크래핑 데이터 (DB에 영속화하지 않음)
- 차트 데이터는 단기 캐시, 장르 아티스트는 장기 캐시
- 스크래핑 실패 시 빈 결과로 fallback

## 추천 엔진 (Recommendations)

사용자 행동 기반 개인화 추천. 두 전략을 **병렬 실행**.

### Channel-based 추천 (`fromChannels`)

사용자가 자주 듣는 채널의 영상을 추천.

1. 최근 재생 기록 + 좋아요에서 채널별 재생 빈도 집계
2. 상위 3개 채널 + 모든 팔로우 채널을 검색어로 사용
3. 각 채널명으로 YouTube 검색
4. videoId 중복 제거 후 최대 5개 결과

### Recent-based 추천 (`fromRecent`)

최근 재생한 영상과 관련된 영상을 추천.

1. 최근 재생한 3개 영상을 기준으로 선택
2. 모든 재생 기록 + 좋아요를 제외 집합으로 구성 (이미 아는 곡 제외)
3. 각 기준 영상의 관련 영상을 YouTube에서 조회
4. 제외 집합 필터링 후 최대 5개 결과

### 공통 정책

- 두 전략은 독립적으로 실행, 개별 실패는 조용히 무시
- 전체 실패 시 빈 결과 반환

## 온보딩 (Onboarding)

신규 사용자의 초기 장르/아티스트 선택 및 라이브러리 시딩.

### 신규 사용자 판단

- **재생 기록 개수 === 0 AND 좋아요 개수 === 0** 인 사용자만 온보딩 대상
- 한 번이라도 재생하거나 좋아요하면 온보딩 불필요

### 플로우 (2단계)

1. **장르 선택**: 멜론 장르 차트에서 아티스트 목록 표시. 최소 2개 장르 필수
2. **아티스트 선택**: 선택한 장르의 인기 아티스트 중 최소 3명 선택 필수
3. **건너뛰기**: 언제든 가능 → 홈으로 이동

### 시딩

선택한 아티스트를 기반으로 초기 라이브러리 구성:

- 각 아티스트당 최대 3개 트랙을 **좋아요**로 추가
- 각 아티스트의 첫 번째 트랙 채널을 **팔로우**
- 시딩된 데이터는 일반 사용자 데이터와 동일하게 취급

## 홈 화면 (Home)

사용자에게 종합적인 음악 탐색 경험을 제공하는 집계 화면.

### 구성 요소

| Section | Source | Description |
|---------|--------|-------------|
| 실시간 차트 | Melon | 상위 5곡 |
| HOT100 | Melon | 상위 5곡 |
| 일간 차트 | Melon | 상위 5곡 |
| 최근 재생 | History | 최근 6곡 |
| 좋아요 수 | Likes | 전체 카운트 |
| 추천 | Recommendations | 채널 기반 + 최근 기반 (선택) |

### 정책

- 모든 데이터를 병렬로 조회
- 외부 소스(Melon) 실패 시 빈 결과로 fallback
- 추천이 없으면 해당 섹션 미표시
