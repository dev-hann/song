# History Domain

## 재생 기록 (History)

사용자의 오디오 재생 이력. 재생 시 자동 기록.

### 속성

| Property | Description |
|----------|-------------|
| id | 고유 식별자 (자동 생성) |
| videoId | YouTube video ID |
| title | 영상 제목 (재생 시점 스냅샷) |
| channel | 채널명 (재생 시점 스냅샷) |
| thumbnail | 썸네일 URL (재생 시점 스냅샷) |
| duration | 재생 시간 (초, 재생 시점 스냅샷) |
| playedAt | 재생한 시각 |

---

## 비즈니스 정책

### 자동 기록

- 트랙 재생 시 클라이언트에서 자동으로 추가
- 사용자가 명시적으로 기록을 추가하지 않음

### Upsert (갱신)

- 동일 영상을 다시 재생하면 **기존 항목의 `playedAt`만 갱신** (새 항목 생성 안 함)
- 효과: 최근 재생한 곡이 목록 최상단으로 이동

### Pruning (가지치기)

- 재생 기록이 **200개를 초과**하면 가장 오래된 것부터 삭제해 **100개**로 축소
- 트리거: 기록 추가 시마다 초과 여부 확인
- 목적: 무한 증가 방지, 최근 기록 중심 유지

### 삭제

- **전체 삭제만 지원** (개별 삭제 불가)
- 전체 삭제 시 사용자의 모든 재생 기록 제거

### 순서

- 항상 `playedAt` 내림차순 (최근 재생한 순)

---

## 온보딩 판단

- 신규 사용자 판단 기준: **재생 기록 개수 === 0 AND 좋아요 개수 === 0**
- 두 조건 모두 0이어야 온보딩 필요
- 한 번이라도 재생하거나 좋아요하면 온보딩 불필요로 전환
