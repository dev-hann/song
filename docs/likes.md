# Likes Domain

## 좋아요 (Like)

사용자가 저장한 트랙. 개인 음악 라이브러리의 기본 단위.

### 속성

| Property | Description |
|----------|-------------|
| videoId | YouTube video ID. 사용자당 고유 |
| title | 영상 제목 (추가 시점 스냅샷) |
| channel | 채널명 (추가 시점 스냅샷) |
| thumbnail | 썸네일 URL (추가 시점 스냅샷) |
| duration | 재생 시간 (초, 추가 시점 스냅샷) |
| likedAt | 좋아요한 시각 |

### 식별자

- **복합 키**: `(userId, videoId)` — 사용자는 동일 영상을 하나만 좋아요할 수 있음

---

## 비즈니스 정책

### Upsert (추가/갱신)

- 동일 영상을 다시 좋아요하면 **기존 항목 갱신** (새 항목 생성 안 함)
- 갱신 시: 메타데이터(title, channel, thumbnail, duration) 업데이트 + `likedAt` 초기화
- 목적: 사용자가 동일 곡을 여러 번 눌러도 항상 하나만 유지

### Track Snapshot

- 좋아요 추가 시점의 YouTube 메타데이터를 **스냅샷으로 저장**
- 이후 YouTube 원본이 변경되어도 로컬 데이터는 유지
- 이유: 사용자가 저장한 시점의 정보를 보존

### 삭제

- 명시적 삭제만 가능 (토글 아님)
- 존재하지 않는 좋아요 삭제 시 오류 반환

### 순서

- 항상 `likedAt` 내림차순 (최근 좋아요한 순)

## 클라이언트 정책

### Optimistic Update

- 좋아요 토글 시 **즉시 UI 반영** (서버 응답 대기 없이)
- 서버 요청 실패 시 **이전 상태로 롤백**
- 좋아요 목록 캐시와 좋아요 여부 캐시를 동시에 업데이트

### 온보딩 시딩

- 온보딩 완료 시 선택한 아티스트당 최대 3개 트랙을 좋아요로 추가
- 시딩된 좋아요는 일반 좋아요와 동일하게 취급
